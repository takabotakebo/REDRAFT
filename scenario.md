# シナリオ設計 — RE-DRAFT

---

## 1. 世界観設定

### 舞台

架空の「ある夜」。  
作者が、締切に追われながら小説を書いている。  
使用ツールは、Googleドキュメント風の共同編集ドキュメント。  
ある時点で共有リンクが外部に流出し、プレイヤーがそれを見つけた。

作者は気づいていない——はずだった。

### 世界の前提

- ゲームは1ページのウェブ画面で完結する
- プレイヤーが見ているのは「流出した共有ドキュメント」
- 作者はリアルタイムで小説を書いている
- プレイヤーには「指摘する」ボタンのみが与えられている
- 指摘すると、原稿は最後の正常地点まで巻き戻る

### 架空組織・固有名詞

| 項目 | 内容 |
|---|---|
| ドキュメントタイトル（初期） | 新作小説_第一稿 |
| 作者名 | 表示なし（「作者」のみ） |
| 編集者名 | プレイヤーが「担当編集者」として扱われる |
| ゲーム内小説タイトル | 「切符」（仮）※ 作中でも未確定のまま扱う |

---

## 2. 裏側の真相

### 表向きの構造

「流出した原稿を覗き見しながら、変な箇所を指摘するゲーム」

### 実際に起きていること

作者は、書けない状態にある。  
生活が崩れ、睡眠が取れず、誰かに見られている感覚があり、締切が迫っている。  
それでも書き続けている。書くことが、今の自分を保つ最後の手がかりだから。

プレイヤーの「指摘（巻き戻し）」は、作者を正常に戻しているように見えて、同時に「何度も同じ文章を書き直させている」行為でもある。

終盤、作者はその事実に気づき、怒りを向け、救難を求め、停止する。

### プレイヤーが最終的に知ること

- 自分は「助けていた」のか「苦しめていた」のか、判断できない
- 小説が完成することが、作者の救済なのか終わりなのか、分からない
- 「流出した共有リンク」という設定は、最初から「誰かに見てほしかった」ということかもしれない

### エンディングの確定

**案Aを採用。案Cを後日談として重ねる複合エンディングとする。**

完成後、コメント欄に作者からの最後の返信が出る：

```
あなたは、編集者ではありませんよね。
でも、助かりました。
```

その直後、ドキュメントのタイトルが一瞬だけ変わる：

```
完成稿  →  次の原稿
```

これはプレイヤーが「本当に終わったのか」を迷う여余地を作るための演出。  
案Bの「小説の中に巻き戻りの記憶が入っている」要素は、ステージ8の本文に小さく仕込む（後述）。

## 3. 登場人物・組織

### 作者

名前・性別・年齢：不明。ドキュメント上には一切表示されない。  
状態の変化：

| 段階 | 状態 |
|---|---|
| 段階1 | 少し変な人。ドジな作家。返信は自虐的で軽い。 |
| 段階2 | 疲れている。本音が漏れる。まだ笑いに変換している。 |
| 段階3 | 見られていることに気づき始める。不安が混じる。 |
| 段階4 | 画面の書式が崩れる。コントロールが失われている。 |
| 段階5 | 文章の意味が繋がらなくなる。夢と現実が混ざる。 |
| 段階6 | 反復が止まらない。言葉が呪詛のように連続する。 |
| 段階7 | プレイヤーへの怒りが明確になる。「戻す人」と認識される。 |
| 段階8 | 救難。停止。最後の返信。 |

### ミナト（ゲーム内小説の主人公）

中学生。性別明示なし。  
夏休みの夜、一人でいる。母を待っている。  
夢の中で切符を持ち、「どこへでも行ける」と言われる。  
選んだ行き先は「お母さんのところ」。  
ミナトの内面は、作者自身の状態と少しずつ重なっていく（孤独、待つこと、言えなかった言葉）。

### プレイヤー

流出した共有リンクを入手した何者か。  
作者には「担当編集者」として認識されている。  
実際には編集者ではないが、それを訂正する手段がない（ボタンは一つしかない）。

## 4. ページ別シナリオ（8段階の役割）

| 段階 | 対応する小説パート | 主な異変カテゴリ | 狙い |
|---|---|---|---|
| 1 | 夜の部屋・ミナトが眠るまで | ギャグ・脱線系 | 「変な作家のドジ」と思わせる |
| 2 | 知らない駅で目を覚ます | 本文混入系・書式変更系 | 作者の疲れが滲み始める |
| 3 | 切符と駅員のやりとり | UI干渉系・見られている感覚 | 「誰かが読んでいる」を意識させる |
| 4 | 切符に書く行き先を迷う | レイアウト崩壊系・書式変更系 | 視覚的な崩れが増える |
| 5 | 今年の春・父が出ていった日 | 意味崩壊・夢文系 | 文章の意味が繋がらなくなる |
| 6 | 電車に乗る・弁当屋に着く | 反復呪詛系・画像貼付系 | 言葉が呪詛のようになる |
| 7 | 母との会話・車に乗る | 怒り・責任転嫁系 | プレイヤーへの怒りが明確になる |
| 8 | ラスト・ミナトの顔 | 救難系・小説と作者の重なり | 完成させてよいのか迷わせる |


## 6. ヒント候補・キーワード候補

### ゲーム内で意味を持つキーワード

- 「切符」：どこへでも行ける、行き先を自分で書く——作者が「出口」を求めている暗示
- 「お母さんのところ」：最も遠くではなく、最も近い人を選ぶ——作者の救難の方向と重なる
- 「たぶん」：ミナトが使う言葉。確信のない、でも踏み出す言葉。エンディングに2度使われる。
- 「戻る」：巻き戻し行為と、小説のテーマ（過去に戻りたいか）が重なる
- 「止まったままの時計」：段階1で出てくる。段階が進むと別の意味に見える。

### ARG的な仕込みの候補（arg-design.mdで詳細化）

- コメント欄に出る作者返信の「時刻表示」はプレイヤーの現在時刻で表示される（リアルタイム感の演出）
- ドキュメントのHTMLソースに作者からのメッセージが仕込まれている
- 白文字テキスト（段階4）を選択すると隠しテキストが読める

## 7. Web演出案

### 基本UI演出

| 演出 | タイミング | 内容 |
|---|---|---|
| タイトル変化 | 異変時・エンディング時 | ドキュメントタイトルが変わる |
| 保存状態変化 | 異変時 | 「保存しています…」→「保存できません」 |
| 閲覧者数 | 常時固定 | 「閲覧者 1」で固定（変化しない） |
| コメント欄異常 | 段階3以降 | 作者返信だけが先に出る、赤文字化、一瞬消える |
| カーソル点滅停止 | 段階8最終停止時 | 執筆停止を視覚的に示す |
| タイピング速度変化 | 段階に応じて | 疲労→遅い、反復呪詛→異常に速い、救難→途切れ途切れ |

### 隠し演出候補

- **白文字テキスト**：段階4で白文字の文が本文に入る。選択しないと読めない。
- **コメント欄の時刻**：コメント追加時の現在時刻を表示（リアルタイム感を演出）
- **保存状態の細かい変化**：「保存しています…（12回目）」など
- **HTMLソースへの仕込み**：ページソースに作者からのメッセージが埋め込まれている

### エンディング演出の詳細

1. 最後の文章がタイピングされ終わる
2. ヘッダーが「すべての変更を保存しました」になる
3. 2秒後、コメント欄に作者の最後の返信が出る（ゆっくりフェードイン）
4. 5秒後、ドキュメントタイトルが「完成稿」に変わる
5. さらに3秒後、タイトルが「次の原稿」に変わる
6. 画面が静かになる（タイピング音が消える）
7. ボタンが消える（指摘ボタン・リセットボタン両方）
8. ドキュメントが「閲覧専用」表示になる

## 8. 次工程への申し送り

### ARG設計担当へ

- ARGとしての入口はintro.html（ブログ記事風）で設計済み（arg-design.md参照）
- コメント欄の時刻は現在時刻で表示（固定値なし）
- 閲覧者数は「閲覧者 1」固定（変化なし）
- 「次の原稿」タイトル変化はエンディングの余韻演出として使用。外部への誘導はしない。

### 実装担当へ

- 小説本文を8ステージ・各複数セグメントに分割した（このscenario.mdが設計図）
- 各ステージに異変テキスト3〜4種を定義済み
- 作者返信文も段階ごとにトーンが変わる設計にしてある
- 段階7・8の怒り系・救難系は「異変がない時に指摘した場合」の返信文も変える必要あり（別設計）
- 白文字テキスト（段階4）はCSS `color: transparent` または `opacity: 0` で実装し、選択時に読める仕様
- 音響：タイピング速度と音の変化は `useTypingEngine` で制御する想定

### タイトル検討について

仮タイトル「流出原稿（仮）」のまま進めているが、正式タイトルは別途決定が必要。  
候補メモ（原案より）：「書きかけの夜」「これは小説ではない」「終わらない一行」「誰かが続きを書いている」  
ゲーム内小説のタイトルは「切符」（仮）として、作中でも未確定のまま扱う。
