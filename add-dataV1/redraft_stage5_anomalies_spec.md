# RE:DRAFT 第5段階 異変仕様書

## 位置づけ

第5段階は、ラストに発生する固定イベントです。

ここまでの段階では、小説本文の崩れ、書式の異常、作者の精神状態の乱れが少しずつ見えていました。  
第5段階では、小説の内容そのものではなく、**作者側の現実**が原稿画面に漏れ出します。

作者は、担当編集である **山島さん** をすでに殺している。  
しかし作者本人は、その事実を忘れている、あるいは思い出さないようにしている。  
プレイヤーが赤字を入れ、異変を指摘し続けることで、作者の中に封じ込められていた記憶がこじ開けられていきます。

第5段階では、以下の3つの異変が**必ずすべて発生**します。  
ランダム出現ではなく、クライマックス用の固定イベントとして扱います。

---

## 第5段階で必ず発生する異変

1. **A5-01：原稿画面が事件現場に変わる**  
2. **A5-02：山島さんのスマホが鳴り続ける**  
3. **A5-03：作者が山島さんへの返信を書き始める**

発生順もこの順番を推奨します。

> 記憶が漏れる  
> → 証拠のようなものが現れる  
> → 作者が山島さんに向けた文章を書き始める

という流れにすることで、作者が「思い出してしまう」過程が自然に伝わります。

---

# A5-01：原稿画面が事件現場に変わる

## 出現位置

以下の本文付近で発生します。

```text
ミナトは店の隅の椅子に座った。
母は黙って片付けを続けた。そのひとつひとつを、ミナトは見ていた。
```

## 異変の概要

小説内の「母が店を片付ける」描写が、作者の現実の「事件現場を片付けた記憶」にすり替わっていきます。

最初は通常の小説本文として進みます。  
しかし、途中から作者自身の記憶が割り込み、母の片付けではなく、作者が玄関で血を拭いていた記憶へ変わります。

この異変で、作者は山島さんを殺したことを思い出しかけます。

## 表示演出

まず、正常本文が入力されます。

```text
ミナトは店の隅の椅子に座った。
母は黙って片付けを続けた。
```

その直後、画面にノイズが入り、本文が一瞬だけ薄くなります。  
その下に、別の文章が割り込むように入力されます。

```text
私は玄関の隅に座っていた。
山島さんは、床に倒れていた。
```

すぐにその2行へ取り消し線が入ります。

```text
~~私は玄関の隅に座っていた。~~
~~山島さんは、床に倒れていた。~~
```

そして消えます。

また正常本文に戻ります。

```text
そのひとつひとつを、ミナトは見ていた。
```

しかし、次の瞬間に書き換わります。

```text
そのひとつひとつを、私は拭いていた。
```

ここから赤字で入力されます。

```text
床の血は、思ったより乾くのが遅かった。
だから原稿を開いた。
書けば、別のことを考えられると思った。
```

少し間を置いて、画面全体に短いノイズが入ります。  
そのあと、作者が思い出すように、短い文が断続的に表示されます。

```text
忘れていた。

そうだ。

そうだった。
```

さらに続きます。

```text
山島さんは、もう帰ったんじゃない。
帰れなくなったんだ。
```

最後に、本文のフォントが少し崩れ、プレイヤーへ向かいます。

```text
でも、じゃあ。

さっきから赤字を入れているアンタは誰なんだ？
```

## 演出意図

この異変では、小説内の「母の片付け」と、作者の現実の「事件現場の片付け」を重ねます。

プレイヤーには、以下のことが伝わるようにします。

- 作者は山島さんを殺したことを忘れていた
- 原稿を書いている途中で、その記憶が戻ってきた
- 山島さんはもう赤字を入れられる状態ではない
- それなのに今も原稿に指摘が入っている
- 作者は、プレイヤーの正体に疑いを向け始める

最後の問い、

```text
さっきから赤字を入れているアンタは誰なんだ？
```

は、第5段階全体の大きな転換点になります。

# A5-02：山島さんのスマホが鳴り続ける

## 出現位置

以下の本文付近で発生します。

```text
仕事を終えた母と、車に乗った。
ミナトはポケットに手を入れる。
```

## 異変の概要

小説の「母と車に乗る」場面が、作者の部屋にある山島さんのスマホの記憶に侵食されます。

山島さん本人からメッセージが来るわけではありません。  
山島さんのスマホが作者の部屋にあり、仕事関係者からの着信が鳴り続けている、という異変です。

作者は、その着信に苛立ち、恐怖し、最後には山島さんのスマホの電源を切ります。

## 表示演出

まず、正常本文が入力されます。

```text
仕事を終えた母と、車に乗った。
ミナトはポケットに手を入れる。
```

その直後、カーソルが止まります。  
数秒後、同じ位置に別の文章が入力されます。

```text
机の上で、スマホが鳴っていた。
```

この一文が一度消えます。

```text
仕事を終えた母と、車に乗った。
```

再び書かれます。

```text
机の上で、スマホが鳴っていた。
山島さんのスマホだった。
```

ここから、通知ログのように本文中へ着信表示が連続で挿入されます。

```text
着信：編集部 佐々木
着信：営業部 井上
着信：山島 自宅
着信：非通知
着信：編集部 佐々木
着信：編集部 佐々木
```

通知音のような短いテキストも混ざります。

```text
ブッ
ブッ
ブッ
```

さらに文章が続きます。

```text
鳴るたびに、画面が光った。
光るたびに、部屋の隅が見えた。
見えない方がよかった。
```

ここまでが一度、すべて選択状態になります。  
青く反転したあと、消えます。

すぐに赤字で再入力されます。

```text
誰かが探している。
誰かが山島さんを探している。
山島さんはここにいるのに。
ここにいるのに、出ない。
出られない。
```

次の文は、通常文字で入力されます。

```text
私はその音が嫌になって、スマホを裏返した。
まだ鳴った。
布の下に入れた。
まだ鳴った。
引き出しにしまった。
まだ鳴った。
```

最後は短く、淡々と残します。

```text
だから、電源を切った。
```

数秒間、カーソルだけが点滅します。

その後に小さく表示されます。

```text
静かになった。
静かになったのに、まだ鳴っている気がした。
```

## 演出意図

この異変では、山島さんの不在を直接説明しません。

かわりに、

- 山島さんのスマホが作者の手元にある
- 仕事関係者からの着信が鳴り続けている
- 山島さんは応答できない
- 作者はそれに耐えられず、電源を切った

という断片から、山島さんがすでに帰れない状態になっていることを匂わせます。

「電源を切った」は、山島さんをさらに黙らせる行為にも見えるため、かなり強いホラー演出になります。

# A5-03：作者が山島さんへの返信を書き始める

## 出現位置

以下の本文付近で発生します。

```text
あの時書いた行き先は、「お母さんのところ」。
```

## 異変の概要

小説の結末が、作者から担当編集・山島さんへの返信文に乗っ取られます。

最初は謝罪メールのように見えます。  
しかし、だんだん事件当日の記憶が漏れ、最後には「殺したのは山島さんの方だ」という歪んだ責任転嫁に変わります。

## 表示演出

まず、正常本文が入力されます。

```text
あの時書いた行き先は、「お母さんのところ」。
```

その直後、文章の下にメール本文のようなブロックが入力されます。

```text
山島さんへ

先ほどはすみませんでした。
言いすぎました。
赤字の件も、ちゃんと直します。
直しますから、もう一度だけ読んでください。
```

一度、最後の行だけ消えます。

```text
直しますから、もう一度だけ読んでください。
```

消えたあと、別の文で書き直されます。

```text
直しますから、返事をしてください。
```

さらに続きます。

```text
返事がないのは分かっています。
でも、既読がつかないのはおかしいです。
あなたのスマホはここにあります。
机の上にあります。
```

「ここにあります。」が何度か増えます。

```text
ここにあります。
ここにあります。
ここにあります。
```

途中でノイズが入り、本文が一瞬だけ乱れます。  
そのあと、事件当日の記憶に寄ります。

```text
どうして玄関に来たんですか。
今日は来ない約束だったじゃないですか。
私はまだ直している途中だったんです。

あなたがまた赤字を入れるから。
まだ読まないでって言ったのに。
```

ここから赤字に変わります。

```text
ごめんなさい。
ごめんなさい。
ごめんなさい。
```

続けて、文字がやや大きくなります。

```text
でも、最初に殺したのはあなたです。
私の原稿を。
```

最後に、この2行だけ残り、それ以前の謝罪文が薄くなります。

```text
でも、最初に殺したのはあなたです。
私の原稿を。
```

## 演出意図

作者は山島さんに謝っているように見えます。  
しかし、文章が進むにつれ、

- 返事がない
- スマホがここにある
- 玄関に来た
- まだ読まないでと言った
- 赤字を入れられた

という断片が漏れていきます。

終盤の、

```text
でも、最初に殺したのはあなたです。
私の原稿を。
```

は、作者の責任転嫁です。

作者は山島さんを殺した現実を、山島さんが「原稿を殺した」ことへの報復のように歪めています。

---

# 第5段階の発生順まとめ

第5段階では、この3つの異変が**必ずすべて発生**します。

## 1. A5-01：原稿画面が事件現場に変わる

小説の「片付け」描写から、事件現場の記憶が漏れ始めます。  
作者は、山島さんを殺したことを思い出しかけます。

## 2. A5-02：山島さんのスマホが鳴り続ける

山島さんのスマホが作者の手元にあることが分かります。  
仕事関係者からの着信が鳴り続け、作者は耐えられず電源を切ります。

## 3. A5-03：作者が山島さんへの返信を書き始める

山島さんへの謝罪文が本文を乗っ取ります。  
最後に、作者は

```text
でも、最初に殺したのはあなたです。
私の原稿を。
```

と責任転嫁します。

この順番により、

> 記憶が漏れる  
> → 証拠が見える  
> → 本人への歪んだ返信を書く

という流れになります。

---

# 実装上の注意

第5段階は、通常の「ランダム異変」ではなく、**固定のクライマックス演出**として扱います。

そのため、実装上は以下のように分けるとよいです。

- 通常段階：異変候補からランダムに発生
- 第5段階：A5-01、A5-02、A5-03を順番に必ず発生
- それぞれの異変後に、プレイヤーが指摘できる
- ただし第5段階では、指摘しても完全には正常化せず、次の固定異変へ進む
- 3つすべてを通過した後、最終停止または最終演出へ移行

第5段階は、「プレイヤーが修正しているはずなのに、もう修正では止まらない」という印象を出すのが重要です。
