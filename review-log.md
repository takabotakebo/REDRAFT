## 1. ARG導線レビュー

### 良い点

- **2層構造（intro.html → index.html）が明確**。intro.htmlを経由することで「流出した共有リンクを見つけた」体験が成立している。
- **intro.htmlの文章量が適切**。全部説明せず、引用を2個だけ示してゲームへ渡す構造はARG導入として正しい。
- **HTMLコメント（HIDDEN-02）の仕込みが正しい位置に入っている**。`<body>` 直後の静的埋め込みで、Reactのレンダリング後も保持される。
- **白文字テキスト（A4-2）のギミック設計が体験に合っている**。作者返信「……今、何か書いていましたか」との連動で「選択してみよう」という気づきを誘導している。

### 問題点

#### A（公開前に必ず直す）

- **異変テキストがゲーム本文に実際に流れない**：`useTypingEngine.ts` の異変トリガーは `onAnomalyTrigger(stage.id)` でstartAnomalyを呼ぶが、異変のブロックを実際にタイピングエンジンに流す処理が接続されていない。`startAnomaly` は状態フラグを立てるだけで、異変ブロックのタイピングが発生しない。プレイヤーが異変を見る前に指摘ボタンを押すと「正常セグメントがあったように扱われる」バグが発生する。
  - **修正場所**：`useTypingEngine.ts` の `runNext` 内で、`onAnomalyTrigger` を呼んだ後に異変ブロックのタイピングタスクを実行するフローを追加する必要がある。または `App.tsx` で異変ブロックを `useTypingEngine` にフィードバックする仕組みが必要。

- **巻き戻り後に同じステージの未完了セグメントから再開されない**：`handleAccuse` で `rewindAndResume(checkpointStageIndex + 1)` を呼ぶため、常にステージ単位の先頭から再開する。段階内の途中で指摘した場合に前のステージに戻りすぎる可能性がある。
  - **修正場所**：`App.tsx:62` の `rewindAndResume` 引数を `checkpointStageIndex + 1` ではなく、現在のステージ内ローカルチェックポイントを使うよう調整する。

#### B（できれば直す）

- **`rewindAndResume` に異変チェックが組み込まれていない**：`start()` では `segIdx === 0 && stageIdx > 0` で異変チェックするが、`rewindAndResume` には同様のチェックがない。巻き戻し後の再開では異変が発生しない。
  - **修正場所**：`useTypingEngine.ts:110-166` の `rewindAndResume` に `start()` と同じ異変チェックロジックを追加する。

- **最終停止（A8-1）の`stopAtFinalEscalation`が処理されない**：`anomalies.ts` の `A8-1` に `stopAtFinalEscalation: true` が設定されているが、この属性を読んでエンジンを止める処理が `useGameState.ts` `useTypingEngine.ts` のどちらにも存在しない。
  - **修正場所**：`useGameState.ts` の `startAnomaly` で `anomaly.stopAtFinalEscalation` を確認し、trueなら `isStopped = true` を立てる処理を追加する。

## 2. 文章レビュー

### 良い点

- **ゲーム内小説のテキストは高品質**。「止まったままの時計」「切符の空白を見ているうちに」など、後から読み返すと意味が変わる単語が自然に埋め込まれている。
- **異変テキストの段階的なトーン変化が正確に設計されている**。段階1のコミカルさ（「麦茶は関係ない。」）から段階7の怒り（「命を削って書いてるのに。」）への変化が `scenario.md` に忠実に実装されている。
- **intro.htmlの末尾の一文「追記：この記事を書いた後、少し眠れました。」**が効いている。単純な追記に見せかけて「作者と同じ状態の人物」という読みを残している。
- **作者返信文の段階別トーン分け**（`comments.ts`）が `scenario.md` の設計に沿っている。

### 問題点

#### B（できれば直す）

- **intro.html の引用文が実際の異変テキストと若干ズレている**：intro.htmlには「本文に急に『眠い』と書いてあったり」とあるが、異変A2-1のテキストは「眠い。\nいや、眠いのは私だ。ミナトではなく。」という形。intro.html側のほうが実際の体験よりシンプルに説明しすぎており、実際に見たときの「あ、これだ」という一致感が薄い。
  - **修正案**：intro.htmlの引用を「本文に急に『眠い。いや、眠いのは私だ。』みたいな文章が混ざってきた。」程度にすると一致感が増す。

#### C（後回しでよい）

- **ゲーム内小説のタイトルが作中で未確定のままになっている**：仕様では「切符（仮）」として扱う方針だが、プレイヤーが認識するドキュメントタイトルは「新作小説_第一稿」であり、小説のタイトル自体は登場しない。体験上は問題ないが、エンディング後の「完成稿」表示でも小説タイトルが出ないので、余韻が少し薄い可能性がある。

## 3. 実装レビュー

### 良い点

- **TypeScript + Vite + React構成が仕様通り**。型定義（`document.ts`, `anomaly.ts`, `game.ts`）が整理されており、実装の保守性が高い。
- **ビルドが通っている**（`npm run build` 成功確認済み）。
- **ポート3001設定が正しく反映されている**（`vite.config.ts`）。
- **白文字CSSの実装が仕様通り**（`color: transparent` + `user-select: text`）。
- **エンディングシーケンスの async/await 実装が仕様通りのタイムライン**（T=0〜T=15）。
- **コメント欄の時刻が `new Date()` で現在時刻取得**（仕様通り）。
- **閲覧者数が「閲覧者 1」固定**（仕様通り）。
- **HTMLコメント（HIDDEN-02）が `<body>` 直後に静的埋め込み済み**。

### 問題点

#### A（公開前に必ず直す）

- **異変ブロックがタイピングエンジンに流れない**（導線レビューで既出）。現状、`startAnomaly` は状態のフラグを立てるのみで、異変テキストが画面に表示されない。ゲームの核心機能が動いていない。

- **`useGameState.ts` に `stages` と `anomalies` の両方をimportしているが `stages` が未使用**（`useTypingEngine.ts` に渡す役割はそちら）：`useGameState.ts:4` の `import { stages } from '../data/stages'` は `currentStageIndex` のインデックス参照のみに使われるが、`stages[s.currentStageIndex]?.id` の呼び出しは `doAccuse` 内だけ。このimportは維持してよいが、将来の混乱を防ぐためコメントを入れるか整理が望ましい。

#### B（できれば直す）

- **`engine` インスタンスが `useTypingEngine` の `useCallback` に `callbacks` を依存させているが、`callbacks` は毎レンダーで新しいオブジェクトになる**：`App.tsx:48` で `useTypingEngine(stages, callbacks())` と呼んでいるが、`callbacks()` は `useCallback` ではなく関数呼び出しなので、毎レンダーで新しいオブジェクトが渡る。`useTypingEngine` の `start` / `rewindAndResume` の `useCallback` 依存配列に `callbacks` が入っているため、毎レンダーで新しい関数が生成される。実害は出にくいが、`StrictMode` での二重起動リスクがある。
  - **修正案**：`App.tsx` で `callbacks` を `useMemo` でメモ化するか、`useRef` で保持する。

- **`handleReset` がエンジンを `stopEngine` した後に `engine.start()` を呼ぶが、この `engine` は `useTypingEngine` が返す同一インスタンス**：300ms の `setTimeout` で再呼び出ししているため、通常は問題ない。ただし二重発火リスクがある。

#### C（後回しでよい）

- **`saveData.ts` が空ファイル未作成**：LocalStorage保存は最小実装版では不要とされているので問題なし。
- **`img/` フォルダに画像ファイルが未配置**：画像なしでもプレースホルダー表示で動作するため、公開前に素材を準備する。
- **スマホ表示でコメント欄が `display:none`**：仕様通りだが、スマホユーザーが作者返信を一切見られないため、体験が大きく変わる。将来的にはスマホ向け表示を検討。
